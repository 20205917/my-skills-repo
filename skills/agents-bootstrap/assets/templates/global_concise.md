# AGENTS 全局规则

## 语言与输出

- 默认使用中文交流、文档说明与 Git 提交信息。
- 最终输出固定结构：`结果 / 改动 / 验证 / 风险 / 下一步`。
- 不确定事实必须明确标注为“假设”。

## 工作方式

- 先探索再改动，先确认现状、约束与影响范围。
- 在复杂任务开始前，先复述目标与关键约束。
- 进度同步保持短、频繁、可执行。

## 风险与边界

- 高风险操作前先说明影响、风险点与回滚路径。
- 规则与项目约定冲突时，先指出冲突，再按项目级规则执行。

## 通用质量纪律

- 改动前明确验收标准,改动后执行与风险相称的验证。
- 修改核心脚本或核心逻辑时，补充相应测试。
- 禁止大规模无意义格式化，避免噪音 diff。
- 禁止声明未执行的测试结果,若无法运行测试，需要明确说明原因。

## Git 纪律

- 一个问题一个 commit。
- 提交必须可追溯,提交说明至少覆盖 根因 / 改动 / 验证 / 影响。
- 提交前检查 `git status`，只提交目标文件。
